# Layout PCB

Generate component placement, autoroute via Freerouting, run DRC, and export review images. Human reviews at the end.

## Prerequisite (human, one-time)

"Update PCB from Schematic" is GUI-only — no CLI or Python API exists for this.

1. Open pcbnew: `uv run poe inspect-asm`
2. Press **F8** (Tools → Update PCB from Schematic)
3. Click **Update PCB** — footprints appear piled at origin
4. Save and close

This only needs to happen once per schematic change. After footprints are on the board, this skill handles everything else.

## Phase 1: Generate Placement

### Step 1: Extract connectivity

Read the netlist from the PCB to understand which components share signal nets:

```bash
python3 -c "
import pcbnew
from collections import defaultdict
board = pcbnew.LoadBoard('cad/cm4_carrier.kicad_pcb')
net_to_refs = defaultdict(set)
for pad in board.GetPads():
    net = pad.GetNetname()
    ref = pad.GetParentAsString()
    if net and net not in ('GND', '+3V3', '+5V', ''):
        net_to_refs[net].add(ref)
edges = defaultdict(list)
for net, refs in net_to_refs.items():
    refs = sorted(refs)
    for i in range(len(refs)):
        for j in range(i+1, len(refs)):
            edges[(refs[i], refs[j])].append(net)
for (a, b), nets in sorted(edges.items(), key=lambda x: -len(x[1])):
    print(f'{a} <-> {b}: {len(nets)} nets: {nets}')
"
```

### Step 2: Read layout_spec.yaml

Get board size, block definitions (which components belong to which functional group), and board origin from `cad/layout_spec.yaml`.

### Step 3: Write pcb_placement.yaml

Using the connectivity graph + block definitions + footprint courtyard sizes, write `cad/pcb_placement.yaml` with intelligent positions:

**Placement strategy:**
- Highly-connected component pairs must be close (minimize signal wire length)
- Power input (connectors) near left board edge for cable access
- Main connector (highest fan-out) centered — most connections radiate from it
- Decoupling caps within 5mm of their IC power pins, 2x2 grid not single row
- Headers on board edges, pins facing outward
- Resistors rotated 90° when in a group (termination, bias networks)
- ICs near their associated connectors and passives
- Component center must be `courtyard_width/2 + 3mm` from board edge minimum
- Stagger components vertically within blocks to create routing channels (don't put everything on the same Y)
- 2mm minimum gap between component courtyards

**Courtyard sizes (mm, from KiCad footprint files):**

| Footprint | Width x Height |
|-----------|---------------|
| PinHeader_1x02 | 3.5 x 6.1 |
| PinHeader_1x04 | 3.5 x 11.2 |
| PinHeader_1x06 | 3.5 x 16.2 |
| PinHeader_1x08 | 3.5 x 21.3 |
| PinHeader_2x20 | 6.1 x 51.8 |
| Fuse_1812 | 5.9 x 3.9 |
| SOT-23 | 3.9 x 3.4 |
| C_1210 | 4.6 x 3.2 |
| C_0805 | 3.4 x 2.0 |
| C_0402 | 1.8 x 0.9 |
| SOIC-8 | 7.4 x 4.9 |
| R_0805 | 3.4 x 1.9 |
| R_0603 | 3.0 x 1.5 |
| QFN-20 | 5.2 x 5.2 |
| USB_Micro-B | 9.4 x 6.5 |

**Format:**

```yaml
# AUTO-GENERATED by /layout-pcb skill
# Manual tweaks preserved until file is deleted and recomputed
board_size: [100, 80]
board_origin: [100, 80]
blocks:
  power:
    components:
      J1: [3, 20, 0]       # x, y (mm from board origin), rotation (degrees)
      F1: [15, 12, 0]
```

### Step 4: Place footprints

```bash
python3 cad/drawing/place_pcb.py
```

This reads `pcb_placement.yaml`, clears any stale traces/zones, and moves footprints to position. Also draws board outline on Edge.Cuts if missing.

## Phase 2: Autoroute

Export DSN, route with Freerouting, import result:

```bash
# Export
python3 -c "
import pcbnew
board = pcbnew.LoadBoard('cad/cm4_carrier.kicad_pcb')
pcbnew.ExportSpecctraDSN(board, '/tmp/cm4_carrier.dsn')
print('DSN exported')
"

# Route (single-threaded — multi-thread is broken)
java -jar ~/tools/freerouting.jar \
  -de /tmp/cm4_carrier.dsn \
  -do /tmp/cm4_carrier.ses \
  -mp 20 -mt 1

# Import
python3 -c "
import pcbnew
board = pcbnew.LoadBoard('cad/cm4_carrier.kicad_pcb')
pcbnew.ImportSpecctraSES(board, '/tmp/cm4_carrier.ses')
board.Save('cad/cm4_carrier.kicad_pcb')
print('SES imported and board saved')
"
```

### Freerouting prerequisites
```bash
# Install if missing
mkdir -p ~/tools && curl -L -o ~/tools/freerouting.jar \
  "https://github.com/freerouting/freerouting/releases/download/v2.0.1/freerouting-2.0.1.jar"
```

## Phase 3: DRC Validation

```bash
uv run poe validate-asm
cat cm4_carrier-drc.rpt
```

Target: **0 errors, 0 unconnected** (warnings OK). Common results:
- `copper_edge_clearance` = component too close to board edge → move inward in `pcb_placement.yaml`
- `starved_thermal` = 0402 pad can't get 2 spokes → set `m_MinResolvedSpokes = 1` in ground pour
- `unconnected_items` = Freerouting missed a trace → adjust placement for better routing channel
- `silk_edge_clearance` = cosmetic, OK to ship

## Phase 4: Visual Review (max 5 iterations)

Export the board as SVG for inspection:

```bash
kicad-cli pcb export svg cad/cm4_carrier.kicad_pcb \
  -l F.Cu,B.Cu,Edge.Cuts,F.Silkscreen \
  --black-and-white --exclude-drawing-sheet \
  --page-size-mode 2 --mode-single \
  -o /tmp/pcb_review.svg
rsvg-convert -w 2400 --background-color white /tmp/pcb_review.svg -o /tmp/pcb_review.png
```

Read the PNG and check:
- All traces connected (no floating ratsnest lines)
- Power traces adequately wide
- No traces crossing board edge
- Components not overlapping
- Logical grouping (power section, comms section, headers)

**If DRC errors or visual issues:**
1. Adjust component positions in `cad/pcb_placement.yaml`
2. Re-run from Phase 1 Step 4 (place → route → DRC → review)

**If clean:** proceed to Phase 5.

## Phase 5: Ground Pour

Add ground copper fill on both layers programmatically:

```python
import pcbnew

board = pcbnew.LoadBoard("cad/cm4_carrier.kicad_pcb")

# Remove existing zones to avoid duplicates
zones = list(board.Zones())
for z in zones:
    board.Remove(z)

gnd_net = board.GetNetInfo().GetNetItem("GND")
bb = board.GetBoardEdgesBoundingBox()
margin = pcbnew.FromMM(0.5)

for layer in [pcbnew.F_Cu, pcbnew.B_Cu]:
    zone = pcbnew.ZONE(board)
    zone.SetNet(gnd_net)
    zone.SetLayer(layer)
    outline = zone.Outline()
    outline.NewOutline()
    outline.Append(bb.GetLeft() + margin, bb.GetTop() + margin)
    outline.Append(bb.GetRight() - margin, bb.GetTop() + margin)
    outline.Append(bb.GetRight() - margin, bb.GetBottom() - margin)
    outline.Append(bb.GetLeft() + margin, bb.GetBottom() - margin)
    board.Add(zone)

# Reason: 0402 pads can't always get 2 thermal spokes
board.GetDesignSettings().m_MinResolvedSpokes = 1

filler = pcbnew.ZONE_FILLER(board)
filler.Fill(board.Zones())
board.Save("cad/cm4_carrier.kicad_pcb")
```

Then re-run DRC: `uv run poe validate-asm`

## Phase 6: Final Export

```bash
uv run poe generate-asm
```

Produces: gerbers, drill files, BOM, CPL, PCB STEP, PCB SVG.

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Freerouting leaves unrouted nets | Adjust placement to create routing channels, re-run |
| Edge clearance violations | Move component inward in pcb_placement.yaml |
| Freerouting hangs | Kill, retry with `-mp 5` |
| Components overlap after placement | Increase spacing in pcb_placement.yaml |
| "Board outline malformed" | Ensure `place_pcb.py` ran (draws Edge.Cuts) |
| Stale traces cause DRC chaos | `place_pcb.py` clears traces automatically |
